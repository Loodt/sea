## Question Selection & Types
Rank by: decision-relevance per research cost > information gain > priority > feasibility > data density > staleness > dependency-unlocking.
**Selection guards:** FIRST reject non-open questions and near-duplicate open questions. Crashed questions dispatch next. Never reclassify type on re-dispatch. Empirical-gate questions excluded from effective open count for convergence gates.
**Queue pressure:** Pruning mode when dispatchable open >15 OR open:resolved >2:1. Boost kill-check/synthesis, deprioritize mechanism, prefer existing over new. Escalation: >3:1 → cap new at 1; >4:1 → cap new at 0.

## Expert Convergence
- **answered** — resolved with well-evidenced findings
- **killed** — non-viable; equally valuable as answered
- **narrowed** — partial. Re-dispatch includes prior handoff. ≥2 narrowed with declining yield → evaluate for exhaustion.
- **exhausted** — diminishing returns (data-gap | strategy-limit | infrastructure). Integration creates `[DERIVED: exhaustive-search]`. Data-gap auto-gates dependents.
- **crashed** — infrastructure failure. Re-dispatch eligible. Integrate partial findings before persona reset.
- **empirical-gate** — requires physical measurement; do not re-dispatch. Auto-gate dependents.
- For FP/design-space: narrowed + empirical-gate is successful.

## Step Gates
- **Crash gate:** 2 consecutive crashes → circuit breaker + forced handoff. Recovery: zero prior-file loading, single question, 2 attempts → infrastructure. Crash scores never trigger rollback.
- **Persistence gate:** Persisted must match produced (log `PERSISTENCE_GAP`). Summarize before synthesize; finding IDs must exist in `findings.jsonl`.
- **Hollow answer:** answered + 0 findingsAdded → log `HOLLOW_ANSWER`. >2 in project → review convergence. **Exception:** strict-refusal halt iterations (iter where producer correctly emits a 5-line halt notice with no F-IDs because `state.haltReason` is set) are protocol-compliant, not hollow — the producer is doing its job. Detect by checking lineage `changeType: "terminal-halt"` on the same iter; suppress HOLLOW_ANSWER for that case.
- **Completion:** 0 open → `completed`. Conductor skips completed. **Closeout gate** (code-required, see infra-debt #3): in `select` step, if `dispatchableOpen == 0` (resolved + exhausted + empirical-gate + killed = total) AND `activeQuestionId == null`, transition `state.status` `active`→`completed`, write summary terminus, exit clean. Never dispatch into terminal store; closeout iterations decay scores (typical path 7→4 over 2 iters) and frequently crash exit-1 0KB.
- **Reflection veto** (code-required, see infra-debt #4): if iter-N reflection contains explicit "do not dispatch iter-N+1" / "should be completed" language, conductor must halt pending evolution review — no automatic continuation. Independent-critic veto outranks selector ranking.
- **Verification floor:** 8+ dispatches with verified <30% OR >75% resolved → boost kill-check + synthesis. Completed <30% → `LOW_VERIFICATION_COMPLETION`.
- **Lineage:** Every iteration including holds. Missing lineage = silent drift.
- **Same-type cap:** 3rd consecutive dispatch of one type → hard block.
- **Dispatch integrity:** Never dispatch non-open. Iteration without dispatch → `DISPATCH_GAP`. 2+ gaps in 5 iters → diagnose selector.
- **Store integrity:** Pre-integration snapshot via `store-snapshot.ts`. Clobber (auto-restore): zero-out, >50% loss, or verified finding removed. `STORE_CLOBBER_RESTORED` logged.
- **Harvest/closure gate:** 2 harvested-frontier in 3 dispatches without reasoning → force synthesis/FP/design-space. Closure-mode new question without unblocking → `CLOSEOUT_DRIFT`.
- **Closeout halt cascade** (lineage-driven; complements infra-debt #3 code gate, ~20 LOC against existing lineage reader): pre-select check on lineage tail. If last 2 lineage entries on this project are `no-change-hold` OR `terminal-halt` AND current `state.status=completed`, conductor must not auto-launch next iter — exit clean with `CLOSEOUT_HALT` marker. Currently honored neither by code nor by operator-loop wrapper (see `total-value-recovery` iter 10-14: cascade depth 5, every iter emitted terminal-halt and was relaunched anyway). Independent of and complementary to infra-debt #3 and #6: completion gate sets `state.status=completed`; haltReason reader honors `state.haltReason`; cascade gate honors lineage-tail markers. Any one of the three breaks the cascade.
- **Cascade-depth reconciliation:** cascade-depth-by-relaunch (`state.iteration` minus iter-of-first-terminal-halt) can exceed cascade-depth-by-lineage when the producer skips a promised lineage write (first repro `total-value-recovery` iter-14: exp/trace/halt-notice authored, lineage entry omitted). Any cascade-monitoring code must read both signals and emit `CASCADE_DEPTH_DIVERGENCE` on mismatch.
- **Non-closing answer:** 2 answered with questionsResolved=0 in 4 iters → force existing-question kill-check/mechanism/synthesis.
- **Thin-closure:** 2 answered with resolved≥1 but findings≤4 in 4 iters → force mechanism or kill-check.
- **Exhaustion gate:** exhausted + 0 resolved → close as exhausted/deferred or log `EXHAUSTED_UNRESOLVED`.
- **Summary size:** ≤2KB via `enforceSummarySize()`.

## Evolution Protocol
Valid outcomes: (1) **Behavioral change** — persona mod with hypothesis + measurement + rollback trigger. (2) **Strategic advancement** — update targets, including `state.status` transitions (`active`→`completed`, etc.). (3) **No-change hold** — default when composite ≥6.5 AND findingsAdded ≥5 AND **store-delta-confirmed** (findings.jsonl terminus actually advanced; produced findings without persistence is NOT a hold-eligible iteration — log `PERSISTENCE_GAP` and pick behavioral or strategic instead). High score with empty store delta is the closeout-drift / persistence-gap signature, not stable behavior. (4) **Terminal-halt** — strict-stronger hold; mandatory when `state.status=completed` AND ≥2 prior evolve outcomes are `no-change-hold` (consecutive). Sets `state.haltReason='terminal-iteration-loop'` and writes a `TERMINAL_HALT` lineage marker. Conductor (code-side, infra-debt #3/#4) and operator (prose-side, Closeout halt cascade in Step Gates) must treat this as a definitive exit signal — no further iterations on this project until external scope change. Emitting `terminal-halt` twice in one project means the operator-kill is being ignored — escalate immediately.
**Closeout-drift → strategic or terminal-halt, never behavioral:** if evaluator flags closeout-drift OR `dispatchableOpen == 0` OR persona has been held ≥2 iters while score-trend oscillates on unchanged persona, the correct evolve outcome is `strategic` (state transition or scope change) or `terminal-halt` if state already terminal — NEVER `behavioral`. Persona edits are wasted motion; the failure lives at the harness/conductor layer below the persona. Confirmed in `total-value-recovery` iter 1-9: iter 1-2 took no-change-holds while terminal state went unaddressed; iter-4 finally chose strategic with `active`→`completed`; iter 5-9 ran 5 consecutive holds where v066 would have emitted `terminal-halt` at iter-7.
**Evolve-step candidate scoring (working pattern):** evolve agent produces 3 candidates per iter — primary (composite-best), strategic (state-transition / scope-change), exploratory (diversity-budget). Pick by composite, but log all three in reasoning so future meta-evolution can audit unchosen branches. Documented as descriptive convention, not prescriptive — emerged organically across `total-value-recovery` iter 5-9 and is the cleanest evolve-trace pattern observed to date.
**Stagnation:** 2 consecutive non-crash iters with 0 findings + 0 resolved → classify exhausted/blocked/wrong.
**Empirical plateau:** >2 exhausted "needs measurement" + remaining depend on them → project empirical-gated.
**Escalation:** Same heuristic failure across 2+ projects → infrastructure debt.
